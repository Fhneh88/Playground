# TaskCLI

Менеджер задач в командной строке - развитие идеи [TodoManager](../TodoManager), но с сохранением на диск. Задачи переживают перезапуск: они хранятся в `tasks.txt` в простом текстовом формате (`<0|1>\t<текст>`).

Архитектура намеренно разделена: `TaskStorage` отвечает только за чтение и запись файла и ничего не знает о командах. `Program.cs` - только диспетчер команд и ничего не знает о формате хранения. Это принцип единственной ответственности на практике.

## Команды

```bash
dotnet run -- add <текст>                 # добавить задачу
dotnet run -- list                        # показать все
dotnet run -- list --only-open           # только невыполненные
dotnet run -- done <номер>               # отметить выполненной
dotnet run -- delete <номер>             # удалить
dotnet run -- edit <номер> <новый текст> # изменить текст
```

## Пример сессии

```text
$ dotnet run -- add Купить продукты
Добавлена задача #1

$ dotnet run -- add Сдать отчёт
Добавлена задача #2

$ dotnet run -- done 1
Задача #1 отмечена как выполненная.

$ dotnet run -- list
1. [x] Купить продукты
2. [ ] Сдать отчёт

$ dotnet run -- list --only-open
2. [ ] Сдать отчёт
```

Попытка обратиться к несуществующему номеру выводит понятное сообщение об ошибке без исключения.
