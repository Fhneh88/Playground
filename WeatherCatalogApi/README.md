# WeatherCatalogApi

API, который объединяет два несвязанных на первый взгляд ресурса: текущую погоду по названию города и справочник категорий. Главная учебная задача - кэширование запросов через Redis.

## Эндпоинты

```text
GET  /weather/{city}    - текущая погода (температура, влажность, описание)
GET  /categories        - список категорий
POST /categories        - создать категорию  { "name": "..." }
```

Погода получается от `wttr.in` - публичный API, не требует ключа. Если город не найден или сервис недоступен - возвращается `404`.

## Кэширование

Запросы идут через `ICachedQueryService`, который оборачивает обращения к БД и внешним API кэшем в Redis. Это снижает количество запросов к `wttr.in` при повторных обращениях к одному городу.

## Запуск

Самый простой способ - Docker Compose, он поднимает API, SQLite volume и Redis одной командой:

```bash
docker compose up --build
```

API будет доступно на `http://localhost:5201`.

Для локального запуска без Docker нужен Redis на `localhost:6379`:

```bash
dotnet run --project WeatherCatalogApi
```

База данных `catalog.db` создаётся автоматически при старте.
